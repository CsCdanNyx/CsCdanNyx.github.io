Unix-Like Windows ( 1 ) MSYS2, MinGW 及 Cygwin

:arrow_up: 回到「電腦設定與環境架設紀錄本」目錄

要架設 Unix-Like 的作業環境，主要使用 MSYS2 ( 內建 Mingw ) 與 Cygwin 還有 Git for Windows ，介面使用 Cmder。

若要使用更完整的 Unix 環境，請去商店安裝 Windows Subsysten for Linyx ( WSL ) ，或是使用 Docker、VirtualBox 。

MSYS2

Msys2 除了提供 Unix-like 的環境以外，可以用 pacman 這個 package manager 來管理與更新軟體。原本已經架好了 Mingw-w64，但發現了這個更加好用，於是改用這個。

這邊的步驟完全參考「Installing GCC & MSYS2 - Github」，裡面寫得非常清楚也非常標準。

安裝 Msys2

首先，下載並安裝 msys2-x86_64。

我安裝於 D:\EnvD\msys64 。 [time=Tue, Mar 12, 2019 4:02 PM]

安裝完畢，在 msys2 的 shell 下達：pacman -Syuu，同步資料庫並更新系統的軟體包 ( 不在預定軟體庫內的「本機」軟體包 )。

pacman -Syu 與 pacman -Syuu 不同，新安裝環境時兩個指令好像都有人用。
-uu 如果本地比遠端新會強迫進行 downgrade 。詳細請 google 或 man。

跑完指令，會提示

這時，重啟 msys2，並再次下達 pacman -Syuu，會將其餘軟體包安裝完畢。

gcc/g++ 環境架設

下達以下指令將 mingw 的 gcc 環境架設起來。

--needed 參數可以避免重新安裝已是最新的軟體。

base-devel 是包含開發所需必要軟體的 package group。這個 group 似乎也包含 base 這個 package group。

加入 PATH

依照「Installing GCC & MSYS2 - Github」所寫，將 D:\...\msys64\mingw64\bin 及 D:\...\msys64\mingw32\bin 按照順序 加入 PATH 之中。之後請務必檢查指令搜尋順序是否正確。

我是 D:\EnvD\msys64\mingw64\bin 及 D:\EnvD\msys64\mingw32\bin 。

然後在 msys64/mingw64.ini 及 mingw32.ini 之中設定 「 MSYS2_PATH_TYPE=inherit 」( 將註解拔掉 )，可以允許繼承系統的 PATH 設定。msys2.ini 的就不要動，保持 msys2 乾淨。

MinGW ( 以下可以不用，直接使用 Msys2 所安裝的即可 )

mingw-w64 official site
MinGW official site、MinGW Sourceforge download site

下載並安裝 「MinGW-w64 - for 32 and 64 bit Windows - Sourceforge」。

網路寫 MinGW-w64 在運行速度上比一般的 MinGW 來的快，更新速度也較快。
見：「What is the difference between MinGW, MinGW-w64 and MinGW-builds? - StackOverflow」、「MinGW、MinGW-w64 与TDM-GCC 应该如何选择？」。

Cygwin

下載並安裝 Cygwin。

我安裝於 D:\EnvD\cygwin64 。 [time=Mon, Mar 11, 2019 11:07 AM]

設定 local package 位置，好像沒有差，下一步。
Choose A download Site 這邊可以透過 Cygwin Mirror Sites 來找尋台灣的鏡像站。這邊我選擇 http://ftp.cse.yzu.edu.tw 。

接下來選擇要安裝的 package ，若選擇全安裝，可以參考這個網站「What is the current full install size of Cygwin? StackOverflow」刪除不必要的套件。

但是一個一個檢查一個一個取消勾選真的超級累，沒有其他方式嗎？
有喔！可以用後面安裝的 cyg-apt 來移除或增加 package 喔~

建議先以 default 完成一次最小的安裝，再重新點擊 setup 執行 package 的安裝。否則如果遭遇如以下問題就要等很久才能測試。

在安裝 package 的時候，進度卡在如下圖 “0/Perpetual“ 的地方，這個查了些資料，照做了但是後續再次安裝其餘套件時還會再次發生，所以目前無解。每次都直接按取消即可。 結果，照了CSDN 上說的，先按取消再按否，魔法就發生了！
看這邊：安装Cygwin时没反应了 - CSDN

看到另一篇說這是只有在 http 的 Mirror 才會發生，於是我換成 ftp 的試試。結果，等了一陣子還真的好了！但真的要等一陣子~

其他資料：Cygwin setup.exe hangs during install Windows 8? How should I continue? - StackOverflow

記得得安裝上 devel category 裡的 gcc、g++、make 套件 ( 非 cygwin 內的 mingw package )。

如若要安裝後面的 apt-cyg ，請記得安裝 lynx 或 wget( apt-cyg 官網是示範 lynx )。

:::warning
這邊我們不將 D:\...\cygwin64\bin 加入 PATH 之中，因為我們同時安裝了 mingw，為確保兩者的 gcc 等不會混用，我們只在 PATH 之中加入 mingw。若要使用 cygwin 的 gcc 等，請使用 Cygwin64 Terminial。
:::

安裝 apt-cyg

Cygwin 沒有自己的 package manager，如若要安裝或更新 package 的話，得用安裝 Cygwin 所下載的 setup.exe 安裝檔。直接點開有 GUI 介面，可以點選升級；或是使用 Windows 的 CMD 呼叫 setup-x86_64.exe -q -P 接一些其他參數及所要安裝的 package，非常麻煩~

但！自從有了第三方的 apt-cyg，一切變得非常舒爽！

不過 apt-cyg 好像沒有 upgrade 選項 …

官網安裝方式使用 lynx，若沒安裝請先使用 setup.exe 安裝，或是找其他安裝方式例如 wget。

接著移除安裝檔 ( 應該不會怎樣吧？ )

這就完成啦~!!

只要下 apt-cyg install <package> 就可以順順的安裝了呢！

發現，只要利用 apt-cyg remove <package> 就可以把那些無用的 package 給移除了！不用再慢慢取消勾選到眼殘了~~~
例如：apt-cyg remove `apt-cyg list gnome`

在更新及安裝之前，記得先下 apt-cyg update 更新 package list 喔！
